Big Data Visualization Methods

Handling Big Data is one of the notable hurdles of the 21st century because of the large volume (Volume), inconsistency of data and multiple formats (Variety), and high computing speed (Velocity). But it is important to analyze Big Data to gain insights into multiple fields and discover research opportunities. Data visualization relies on human cognitive capacities to absorb visual information, making it a solution to facilitate meaningful analysis, accessibility, and interpretation of huge data. The goal is to compare visualization methods to see which of them satisfy all the Big Data criteria.

Treemap

This technique is based on filling hierarchical data starting with a root rectangle which is then divided into smaller rectangles. It does satisfy the volume criteria because the number of data layers depends on the volume of data and can show large volumes by iteratively adding a layer of data for each hierarchy. But this visualization technique can display only two data features: shape for volume and color for grouping so variety cannot be shown. Velocity is also not met as it shows data at one time only so cannot be used for data with time series or historical trends. A major limitation is that it can show hierarchical data only.

Circle Packing

This technique is like a treemap except it uses circles as its main shape. Given that it has the same properties as a treemap, it satisfies the volume criteria only because the number of data layers depends on the volume of data.

Sunburst

This technique is also like a treemap except it uses radius and arc length instead of width and height. When the data changes, only the sector with new data must be changed by changing the radius. With animation, the criteria of data velocity can be shown. Given that it has the same properties as a treemap, it satisfies the volume criteria only because the number of data layers depends on the volume of data. Most humans can perceive it easily compared to the first two methods.

Circular Network Diagram

In this technique, data attributes are put about a circle and connected by curves based on how related they are. The different line width or color saturation is the magnitude of relativeness. With interactions, you can choose the relations you want. Aggregated data can be represented as arc sets so that analysts can retrieve information about object relations. The technique can be applied to massive data volume as data attributes can be put around the circle radius and attribute arc area. Also, more data attributes can be shown near an arc and multiple factors in a diagram and make relations between them. This can make it difficult to get insights but in some cases, with this approach, the analyst can get sufficient information. With this the variety criterion is satisfied. Also, with every change in data, the whole diagram must be changed.

Parallel Coordinates

This technique can support multiple data attributes for different data objects. The data factors that must be analyzed are put on one axis and the corresponding data values of the data object are put on the other axis. A data object is displayed connected by spanning lines which display its context of other objects. Only thick lines are used to show individual data with which the data volume criteria are met. The 2D parallel coordinates can be extended to 3D parallel coordinates in which the axes are equally divided with the focus axis in the center of a circle. A data item is shown as line segments passing via all axes. Several factors can be handled this way for a massive number of data objects in a single screen, so it satisfies data variety criteria. Data velocity can be shown as a factor in 3D coordinates.

Steamgraph

This technique is a variation of a stacked area chart, which is shown around a central axis that causes the flowing shape. In this way, different trends or events can be displayed via a timeline. The method shows each time series along with the sum of each time series as the height of each layer. It supports the data velocity criteria and as it works for large datasets it does satisfy volume but since it is 1D, it cannot satisfy the data variety criteria.
